# config

## 模块职责

- `internal/config` 负责配置结构定义、加载和解析。

## 不负责什么

- 不负责配置文件本身的内容维护，文件本体在仓库根 `config/`

## 子目录索引

- 当前无子目录

## 文件索引

- `config.go`
  - 定义配置结构、读取顺序和环境变量覆盖逻辑。
- `config_test.go`
  - 验证配置加载与覆盖行为。

## 对外入口

- 配置加载函数与配置结构

## 关键依赖

- 被 `main.go` 和 `entry/` 装配阶段消费

## 维护提示

- 新增配置项时，通常需要同步更新配置结构、默认配置文件和测试。
