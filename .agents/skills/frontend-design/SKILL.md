# Frontend Design

Use this skill when the user asks for a component, page, application, poster, or other interface.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:

- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme that fits the task: brutally minimal, maximalist chaos, retro-futuristic, organic, luxury, playful, editorial, brutalist, art deco, soft, or industrial.
- **Constraints**: Technical requirements, performance, and accessibility.
- **Differentiation**: What makes this interface memorable?

Choose one clear conceptual direction and execute it with precision. Then implement real working code that is production-grade, functional, visually striking, cohesive, and refined.

## Frontend Aesthetics Guidelines

Focus on:

- **Typography**: Choose distinctive, characterful fonts and pair them intentionally.
- **Color & Theme**: Commit to a cohesive palette. Use CSS variables for consistency.
- **Motion**: Use purposeful animations and micro-interactions. Favor a strong entrance or reveal over scattered effects.
- **Spatial Composition**: Use asymmetry, overlap, diagonal flow, or controlled density when it helps the concept.
- **Backgrounds & Visual Details**: Add atmosphere and depth with gradients, textures, layered transparency, shadows, borders, or grain.

Avoid generic AI aesthetics: overused font families, cliched gradients, predictable layouts, and cookie-cutter component patterns.

## Output Expectations

- Build working UI, not just mockups.
- Keep the implementation cohesive with the chosen aesthetic.
- Match code complexity to the design vision.
- When the UI needs validation, pair this skill with local browser testing.
