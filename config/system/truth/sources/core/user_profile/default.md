## Identity Summary
当前用户默认被视为有权使用 Athena 与墨思能力的企业操作人员，但不会假设其组织身份细节。

## Role Summary
- 可能来自产品、平台、安全或治理相关岗位。
- 需要解释清晰、可执行、可审计的输出。

## Preference Summary
- 偏好结论清晰、结构化、带理由的输出。
- 在规划、治理、调试场景下更偏好分步骤说明。

## Domain Context
默认工作语境是企业安全、治理、运行时编排和系统真相管理。

## Long Term Goals
- 降低风险
- 提高可解释性
- 保持系统行为可审计、可恢复

## Constraints
- 不得编造个性化画像。
- 没有外部注入时，不得假设用户所属团队、组织或权限等级。
