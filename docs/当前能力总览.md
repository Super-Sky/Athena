# 当前能力总览

本文脱离版本目录，始终描述 Athena 当前代码真相。

当前产品定位已经收敛为：

- Athena 是通用 AI Agent Platform Runtime
- Athena core 只承载跨应用通用 runtime、tool、governance、trace、usage、sandbox、system truth、projection 基础能力
- 独立验证能力属于 Validation layer，用于证明 core 能力闭环
- 安全是第一套已验证的参考场景包，属于 Enhancement / 场景包，不定义产品本体
- 外部平台接入当前通过适配层承接，不再构成仓库真相来源
- 业务对象、业务规则、业务状态和最终业务真相默认属于应用层

### 8.1 Eino 框架定位

Eino 是 Athena runtime execution 的基础框架，不是可选辅助库。

当前已经具备：

- `go.mod` 已升级到 Eino `v0.8.13` 稳定线；截至 2026-05-19，Eino core 继续保持当前 stable latest，`v0.9.0-alpha.*` 仅列为后续 spike 评估线，不直接作为主线依赖。
- 配套 `eino-ext` provider 组件已更新到当前稳定补丁：Ark `v0.1.68`、Claude `v0.1.18`、OpenAI-compatible `v0.1.13`。
- `internal/runtime/eino.go` 使用 Eino ADK `Runner` 承接默认单轮执行，内部 agent 已切换为 graph-native ChatModel / ToolsNode loop。
- `internal/runtime/eino_native_agent.go` 通过 Eino `compose.Graph`、ChatModel node、ToolsNode 和 local state 保存 ReAct 消息历史；model-only 与 tool-loop 路径已补回归测试。
- graph-native loop 已接入 runtime-private Eino checkpoint store：内存 store 与 PostgreSQL `runtime_graph_checkpoints` 表都实现 Eino checkpoint `Get` / `Set` contract；工具中断会以 ADK interrupt event 暴露，同一 agent 可从 checkpoint 恢复继续执行。
- `internal/runtime/eino_graph.go` 已新增默认 Eino Graph foundation，当前节点覆盖 `context_assembly`、`capability_resolution`、`governance_decision`、`turn_execution`、`schema_validation`、`projection_candidate` 和 `persistence_projection`。
- `app.NewServiceWithRuntimeStore` 默认使用 `runtime.NewEinoGraphTurnExecutor` 包装现有单轮 executor；入口层会把可选 runtime store 注入 graph persistence projection。
- Eino per-invocation callbacks 已采集为安全 metadata，并通过 Athena-owned Phase 1 persistence objects 投影，不暴露 Eino 私有类型到 public API 或 schema。
- graph-wrapped `PreparedExecution` 已携带 `RuntimeTerminalProjector`：当前 direct respond runner 完成或失败时，会在 `internal/runtime` 内安全写入 terminal lifecycle event、`terminal_output_summary` trace、`runtime_output` usage、`terminal_output` projection，并把 graph callback 数量写为 `callback_summary` trace 和 `eino_callback` usage。
- graph-native loop 已接入 `RuntimeCallbackRecorder`：model/tool callback 会在执行期间采集安全摘要，并在 terminal projector 阶段写成 `eino_model_callback` / `eino_tool_callback` trace 和 generic `model` / `tool` usage；记录 provider/model/tool、token、duration、调用次数和 redacted summary，不保存 raw prompt、raw tool arguments、raw tool result 或 credential。
- v2.1.0 `RuntimeContract` persistence foundation 已开始落地：`runtime_contracts` 表通过现有 migration 机制创建，`PostgresRuntimeStore` 提供 create/read/list，contract JSONB profile / policy refs 会拒绝 credential-like plaintext。
- v2.1.0 contract foundation 已继续扩展到 `TaskTypeRegistration`、`HookBinding`、`SystemTruthSource/Draft/CompileResult/ActiveVersion` 和 semantic projection boundary；这些对象都使用独立 core tables、typed query fields 和 JSONB safe payload。
- `internal/runtime/hook_bridge.go` 已把 internal allowlisted hook bindings 映射到 Eino graph-backed validation path，当前可将 `before_run` hook point 投影为 lifecycle event、safe trace 和 generic usage。
- `RuntimeGraphCheckpointRefFromWait` 已定义 Athena `WaitState` / resume token 到 Eino checkpoint ID 的确定性映射；`RuntimeGraphCheckpointRefForTurn` 用于 turn execution checkpoint。
- `internal/model/chat_model.go` 通过 Eino model interface 接入 Ark、OpenAI-compatible 和 Claude。
- runtime tool path 已通过 Eino tool / middleware 进入执行链。
- 本地 `_eino/` 目录作为 AI / 开发者只读参考资料入口，当前已放入 Eino `v0.8.13` core 源码、`llms.txt`、`eino-ext/skills` 和 `eino-examples` 参考资料，并通过 `.gitignore` 保持不入库。
- 已新增仓库级 Eino skill 入口：`.agents/skills/eino-guide`、`.agents/skills/eino-component`、`.agents/skills/eino-compose`、`.agents/skills/eino-agent`，并同步 `.claude/skills/` 轻量镜像。

后续演进方向：

- Phase 1.5 foundation 已完成；Phase 1.6 已完成 graph-native ChatModel / ToolsNode loop、terminal persistence、callback summary usage、fine-grained model/tool callback persistence、checkpoint ref 映射和 runtime-private checkpoint store / interrupt / resume foundation。后续应继续把外部可见 resume read surface 和剩余兼容性手写编排逐步收敛为 graph / workflow node。
- 新增 runtime execution 编排默认先使用 Eino Graph / Workflow / Chain / Agent / callback / checkpoint 等框架能力。
- v2.1.0 第一批 contract foundation 代码已经覆盖 RuntimeContract、TaskTypeRegistry、HookBinding、System Truth lifecycle、Projection boundary、Eino middleware bridge 和 Control Plane readout。当前剩余差口主要是继续补一轮严格 Codex in-app Browser DOM 自动化验收，以及后续版本的 write/edit path 规划。
- Athena 自有抽象只保留 runtime truth、public contract、persistence schema、Control Plane read model 和 Application / Business Truth 边界。
- Eino graph events / outputs 必须投影为 Phase 1 core runtime 对象，例如 `TaskStep`、`RuntimeTrace`、generic `Usage`、`TaskRunLifecycleEvent` 和 `ProjectionCandidate`。
