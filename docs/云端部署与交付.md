# 云端部署与交付

## 这份文档解决什么问题

这份文档描述 Athena 从代码到服务器的完整交付路径，覆盖：

- Docker 镜像构建
- 单仓 Docker Compose 云端部署
- 一键上传并远端启动
- Linux 二进制打包
- 二进制 + systemd 部署

当前推荐顺序是：

1. Docker Compose 部署
2. 需要无 Docker 运行时或更轻交付时，再走二进制部署

## 仓库内已经提供的交付物

- 后端镜像：
  - `Dockerfile`
- 前端镜像：
  - `web/Dockerfile`
- 单仓 Compose：
  - `deploy/docker-compose.cloud.yml`
- 云端环境变量模板：
  - `deploy/athena.env.example`
- 二进制 systemd 模板：
  - `deploy/systemd/athena.service.example`
- 二进制打包脚本：
  - `scripts/build_release_bundle.sh`
- 一键远端部署脚本：
  - `scripts/deploy_cloud.sh`

## 部署模式选择

### 模式 A：Docker Compose

适合：

- 内部研发环境
- 测试环境
- 一台云服务器快速起服务

特点：

- 路径最短
- 自带 PostgreSQL
- 可选带 `athena-web`

### 模式 B：二进制 + systemd

适合：

- 目标环境统一使用 systemd
- 希望把 Go 二进制作为交付物
- 需要更明确地拆开镜像构建和服务启动

特点：

- 交付包更轻
- 回滚依赖 tar 包和 systemd
- 需要宿主机自己准备 PostgreSQL

## Docker Compose 部署

### 1. 准备环境文件

```bash
cp deploy/athena.env.example deploy/athena.env
```

至少改这些值：

- `POSTGRES_PASSWORD`
- `SECURITY_ENCRYPTION_KEY`
- `CONTROL_PLANE_ALLOWED_ORIGINS`
- 如果要开启 Web 控制台：
  - `ATHENA_WEB_PORT`
  - `COMPOSE_PROFILES=control-plane`

### 2. 本地构建镜像

后端：

```bash
docker build -t athena:local .
```

如果需要 Web 控制台：

```bash
docker build -t athena-web:local -f web/Dockerfile web
```

### 3. 本机验证 Compose

```bash
docker compose --env-file deploy/athena.env -f deploy/docker-compose.cloud.yml up -d
```

如果需要 Web 控制台：

```bash
COMPOSE_PROFILES=control-plane docker compose \
  --env-file deploy/athena.env \
  -f deploy/docker-compose.cloud.yml \
  up -d
```

### 4. 验证服务

```bash
curl http://127.0.0.1:8090/healthz
curl http://127.0.0.1:8090/swagger/openapi.json
```

如果开启 Web：

- `http://127.0.0.1:5173`

### 5. Athena 镜像当前的健康检查方式

Athena 后端镜像当前不再依赖容器内 `curl`。

Compose 中的 healthcheck 直接执行：

```bash
/app/athena healthcheck
```

这个命令会按当前环境配置探测：

```text
http://127.0.0.1:${HTTP_PORT}/healthz
```

这样可以避免因为 runtime 镜像额外安装 `curl` 而引入 `apt-get` 构建耗时。

## 一键部署到云端

仓库已经提供 `scripts/deploy_cloud.sh`，它会：

1. 本地构建 Athena 镜像
2. 可选构建 `athena-web` 镜像
3. 打包 `docker-compose.cloud.yml` 和 `athena.env`
4. 通过 `scp` 上传到目标服务器
5. 在远端执行 `docker load`
6. 在远端执行 `docker compose up -d`

### 使用前提

本地需要：

- `docker`
- `ssh`
- `scp`

远端服务器需要：

- Linux
- `docker`
- `docker compose`

### 执行方法

```bash
cp deploy/athena.env.example deploy/athena.env
```

填写完配置后执行：

```bash
DEPLOY_USER=root \
DEPLOY_HOST=203.0.113.10 \
DEPLOY_DIR=/opt/athena \
scripts/deploy_cloud.sh
```

如果同时部署控制面：

```bash
DEPLOY_USER=root \
DEPLOY_HOST=203.0.113.10 \
DEPLOY_DIR=/opt/athena \
INCLUDE_WEB=true \
scripts/deploy_cloud.sh
```

### 脚本默认约定

- 远端部署目录默认：
  - `/opt/athena`
- 远端 Compose 使用：
  - `docker-compose.cloud.yml`
  - `athena.env`
- 后端默认映射端口：
  - `8090`
- Web 控制台默认映射端口：
  - `5173`

## 二进制打包

### 1. 构建 Linux 发布包

```bash
scripts/build_release_bundle.sh
```

默认输出到：

- `output/release/`

会生成：

- 解包目录
- `.tar.gz` 发布包

发布包中包含：

- `athena` 二进制
- `config/`
- `deploy/systemd/athena.service.example`
- `deploy/athena.env.example`
- `README.md`
- `docs/云端部署与交付.md`

### 2. 上传到服务器

```bash
scp output/release/athena-*.tar.gz root@203.0.113.10:/tmp/
```

### 3. 在服务器解包

```bash
mkdir -p /opt/athena
tar -xzf /tmp/athena-*.tar.gz -C /opt/athena --strip-components=1
```

### 4. 准备环境文件

```bash
cp /opt/athena/deploy/athena.env.example /opt/athena/athena.env
```

根据实际环境填写：

- 数据库地址
- 账号密码
- `SECURITY_ENCRYPTION_KEY`
- `CONTROL_PLANE_ALLOWED_ORIGINS`
- `SHARED_ROOT_DIR`

### 5. 安装 systemd 服务

```bash
cp /opt/athena/deploy/systemd/athena.service.example /etc/systemd/system/athena.service
systemctl daemon-reload
systemctl enable athena
systemctl start athena
```

### 6. 查看状态

```bash
systemctl status athena
journalctl -u athena -f
curl http://127.0.0.1:8080/healthz
```

## 迁移策略

Athena 当前固定要求显式迁移。

Docker 模式：

- 由 Compose 里的启动命令串联：
  - `/app/athena migrate && exec /app/athena api-server`

二进制模式：

- 由 systemd 中的 `ExecStartPre` 执行：
  - `/opt/athena/athena migrate`

## 控制面部署口径

`athena-web` 不是 Node 运行时服务，而是静态站点容器。

因此：

- 开启它时，本质上是在部署一个 `nginx` 容器
- 前提是 `/api` 和 `/swagger` 能反代到 `athena-api`
- 私有化交付默认可以不部署它

## 回滚建议

Docker 模式：

- 保留上一版镜像 tag
- 回滚时重新执行：
  - `docker compose up -d`

二进制模式：

- 保留上一版 tar 包
- 回滚时替换 `/opt/athena/athena`
- 再执行：
  - `systemctl restart athena`
